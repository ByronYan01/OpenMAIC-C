# OpenMAIC 工程介绍（新手快速理解版）

## 1. 这个项目是做什么的
OpenMAIC 是一个“多智能体互动课堂”平台：
- 你输入学习需求（可附 PDF）
- 系统自动生成课程大纲、场景内容、动作脚本
- 在课堂页面里用幻灯片/测验/互动实验/PBL 的方式播放
- 可选生成图片、视频、语音，并支持导出

技术上它是一个 **Next.js 16 + React 19 + TypeScript** 的全栈项目，前后端都在同一个仓库中。

## 2. 先跑起来（最低路径）
前置：Node.js >= 20.9、pnpm >= 10

```bash
pnpm install
cp .env.example .env.local
# 至少配置一个模型 API Key（如 OPENAI_API_KEY / GOOGLE_API_KEY）
pnpm dev
```

打开 `http://localhost:3000`。

## 3. 项目结构怎么读（先看这 6 个位置）

### 3.1 页面入口
- `app/page.tsx`
  - 首页：输入需求、上传 PDF、进入生成预览
- `app/generation-preview/page.tsx`
  - 生成流程可视化页：解析 PDF、生成大纲、生成首个场景
- `app/classroom/[id]/page.tsx`
  - 课堂播放页：加载课堂数据，并继续生成剩余场景

### 3.2 API 入口
- `app/api/generate/*`
  - 拆分式接口：大纲、场景内容、场景动作、TTS、图像/视频
- `app/api/generate-classroom/*`
  - 一次性异步任务接口（提交任务 + 轮询任务状态）
- `app/api/chat/route.ts`
  - 多智能体聊天/讨论相关

### 3.3 核心业务层
- `lib/generation/`
  - 课程生成流水线（大纲生成 + 场景生成）
- `lib/orchestration/`
  - 多智能体编排（LangGraph）
- `lib/playback/`
  - 课堂播放引擎/状态推进
- `lib/server/`
  - 服务端生成任务、落盘存储、任务状态管理
- `lib/store/`
  - 前端 Zustand 状态（stage、settings、media 等）

## 4. 核心实现流程（最重要）

## 4.1 Web 端交互式流程（你在页面里直接体验到的）
1. 首页 `app/page.tsx`
- 保存用户需求、可选 PDF 到 sessionStorage
- 跳转到 `/generation-preview`

2. 预览页 `app/generation-preview/page.tsx`
- 可选：调用 `/api/parse-pdf` 解析 PDF
- 可选：调用 `/api/web-search` 获取检索上下文
- 调用 `/api/generate/scene-outlines-stream` 流式生成大纲
- 先只生成第一个场景（`scene-content` + `scene-actions` + 可选 TTS）
- 把 stage/scenes 写入本地存储，跳转到 `/classroom/[id]`

3. 课堂页 `app/classroom/[id]/page.tsx`
- 从本地存储或 `/api/classroom` 加载课堂
- 通过 `useSceneGenerator` 在后台继续生成剩余场景
- 并行触发媒体生成（图片/视频），逐步补齐内容

## 4.2 服务端异步任务流程（API 一把生成）
1. `POST /api/generate-classroom`
- 创建 jobId
- 记录任务状态（queued）
- 后台启动生成任务

2. `lib/server/classroom-generation.ts`
- 初始化 -> 可选检索 -> 生成大纲 -> 循环生成场景
- 可选媒体生成、可选 TTS
- 最终持久化课堂

3. `lib/server/classroom-job-store.ts`
- 把任务状态写入 `data/classroom-jobs/*.json`（queued/running/succeeded/failed）
- 前端或外部系统轮询 `/api/generate-classroom/[jobId]`

## 5. 数据存储怎么理解
- 前端本地：IndexedDB（stage、scene、音频、图片映射等）
- 会话态：sessionStorage（本次生成参数）
- 服务端任务态：`data/classroom-jobs/*.json`
- 服务端课堂数据：通过 `lib/server/classroom-storage` 统一读写

## 6. 给新手的阅读顺序（30 分钟版）
1. `package.json`：看脚本和技术栈
2. `app/page.tsx`：看入口交互
3. `app/generation-preview/page.tsx`：看生成主流程
4. `lib/hooks/use-scene-generator.ts`：看“剩余场景”如何续生成
5. `lib/server/classroom-generation.ts`：看服务端端到端流水线
6. `lib/server/classroom-job-store.ts`：看异步任务状态管理

## 7. 常见卡点
- 没配模型 Key：生成接口会直接失败
- PDF 解析失败：通常是解析服务未配置或文件异常
- TTS 失败：语音 provider 配置不完整
- 媒体生成慢：图片/视频本身是异步，课堂可先播放文本内容

## 8. 你现在可以怎么继续
- 如果你只想“会用”：先跑通首页到课堂一条链路
- 如果你想“会改”：先改 `app/page.tsx` 的输入项，再看 `generation-preview` 如何把参数传到 API
- 如果你想“会扩展能力”：从 `app/api/generate/*` 和 `lib/generation/*` 开始加新场景能力
