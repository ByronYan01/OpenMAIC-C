# OpenMAIC 前端存储分层说明

> 目标：快速搞清楚这个项目里，哪些数据存在 `localStorage`，哪些存在 `sessionStorage`，哪些存在 `IndexedDB`。

## 1. 先看结论

这个项目前端不是把所有东西都塞进一个地方，而是按数据类型分层存储：

- `localStorage`
  - 存“设置类、偏好类、轻量配置类”数据
- `sessionStorage`
  - 存“本次生成流程临时参数”
- `IndexedDB`
  - 存“课程内容、媒体文件、聊天记录、播放状态”等大体量业务数据

这样设计的原因很简单：

- 设置项体积小，适合直接本地快速读写
- 临时生成参数只在当前会话里有意义
- 场景、音频、图片、视频这类数据体积大，必须放 `IndexedDB`

## 2. localStorage 存什么

核心文件： [settings.ts](C:\D\work\zs\shensi\OpenMAIC\lib\store\settings.ts)

这里存的是全局设置，主要通过 `Zustand persist` 持久化到 `settings-storage`。

典型内容包括：

- 大语言模型配置
  - `providerId`
  - `modelId`
  - `providersConfig`
- 语音配置
  - `ttsProviderId`
  - `ttsModelId`
  - `ttsVoice`
  - `ttsSpeed`
  - `asrProviderId`
  - `asrModelId`
  - `asrLanguage`
  - `ttsProvidersConfig`
  - `asrProvidersConfig`
- PDF 配置
  - `pdfProviderId`
  - `pdfProvidersConfig`
- 图像/视频配置
  - `imageProviderId`
  - `imageModelId`
  - `imageProvidersConfig`
  - `videoProviderId`
  - `videoModelId`
  - `videoProvidersConfig`
- Web Search 配置
  - `webSearchProviderId`
  - `webSearchProvidersConfig`
- 播放和界面偏好
  - `ttsMuted`
  - `ttsVolume`
  - `autoPlayLecture`
  - `playbackSpeed`
  - `sidebarCollapsed`
  - `chatAreaCollapsed`
  - `chatAreaWidth`

所以你前面问的 `ttsModelId/asrModelId`，现在就是和其他模型配置一样，存在这里。

## 3. sessionStorage 存什么

典型文件：

- [page.tsx](C:\D\work\zs\shensi\OpenMAIC\app\page.tsx)
- [page.tsx](C:\D\work\zs\shensi\OpenMAIC\app\generation-preview\page.tsx)
- [page.tsx](C:\D\work\zs\shensi\OpenMAIC\app\classroom\[id]\page.tsx)

这里主要存“本次生成过程的临时参数”，例如：

- `generationSession`
  - 首页填写的 requirement
  - 语言
  - 是否联网搜索
  - PDF 相关上下文
- `generationParams`
  - 从生成预览页跳到课堂页时需要恢复的本次生成参数

特点：

- 只服务当前标签页/当前会话
- 关闭页签后通常就没了
- 适合做页面间临时传参，不适合长期保存

## 4. IndexedDB 存什么

核心文件： [database.ts](C:\D\work\zs\shensi\OpenMAIC\lib\utils\database.ts)

这里存的是“真正的业务数据”和“大文件”。

目前主要表包括：

- `stages`
  - 课程基础信息
- `scenes`
  - 场景内容
- `audioFiles`
  - TTS 音频 blob
- `imageFiles`
  - 图片文件
- `chatSessions`
  - 聊天会话
- `playbackState`
  - 播放进度
- `stageOutlines`
  - 大纲数据
- `mediaFiles`
  - AI 生成的图片/视频文件
- `generatedAgents`
  - 自动生成的智能体角色

相关文件还有：

- [stage.ts](C:\D\work\zs\shensi\OpenMAIC\lib\store\stage.ts)
- [media-generation.ts](C:\D\work\zs\shensi\OpenMAIC\lib\store\media-generation.ts)

这些模块会把课程、场景、媒体任务结果写入 `IndexedDB`，因为它们：

- 数据量更大
- 需要跨刷新恢复
- 可能包含二进制文件（音频、图片、视频）

## 5. 为什么设置不放 IndexedDB

因为“设置”和“业务数据”是两类东西。

设置类数据放 `localStorage` 的好处：

- 初始化简单
- 读取速度快
- Zustand `persist` 可以直接接管
- 不需要单独写数据库表和查询逻辑

如果把模型配置也塞进 `IndexedDB`，会带来几个问题：

- 设置读取链路会变复杂
- 页面初始化时要等异步 DB 读取
- 迁移成本更高
- 会和当前 `settings-storage` 体系冲突

所以当前做法其实是合理的，而且已经和其他模型配置保持一致。

## 6. 你可以怎么理解这套分层

你可以把它简单记成：

- `localStorage` = 用户设置中心
- `sessionStorage` = 本次生成临时上下文
- `IndexedDB` = 课程和媒体资料库

## 7. 和你当前问题直接相关的结论

`TTS/ASR` 新增的模型名配置：

- 已经会持久化
- 持久化位置是 `localStorage -> settings-storage`
- 不需要再额外存一份到 `IndexedDB`
- 这和其他模型配置是一致的
