# OpenMAIC 后端调试手册（按报错关键词定位）

> 用法：你看到报错后，先在下面搜关键词，按“第一定位点 -> 第二定位点 -> 处理建议”排查。

## 1. 通用排查顺序（先背这个）

1. 看 Network 返回的 `code/message`（后端统一 `apiError`）
2. 先看对应 `app/api/**/route.ts` 参数与 header
3. 再看 `lib/server/provider-config.ts` 是否拿到 key/baseUrl
4. 最后看具体 provider 实现（`lib/audio/*` 或 `lib/media/*`）

## 2. 关键词速查

### 2.1 `MISSING_REQUIRED_FIELD`

第一定位点：对应 API route
- `app/api/generate/tts/route.ts`
- `app/api/transcription/route.ts`
- `app/api/generate/image/route.ts`
- `app/api/generate/video/route.ts`
- `app/api/generate-classroom/route.ts`

常见原因：
- 请求体缺字段（如 `requirement`、`text`、`audio`、`prompt`）
- 字段名写错（如 `model` 和 `modelId` 混用）

处理建议：
- 先对照 route 文件里的解构字段名，确保前端参数名一字不差。

### 2.2 `MISSING_API_KEY` / `No API key configured`

第一定位点：`lib/server/provider-config.ts`
第二定位点：对应 route 的 key 解析逻辑
- TTS：`app/api/generate/tts/route.ts`
- ASR：`app/api/transcription/route.ts`
- Image：`app/api/generate/image/route.ts`
- Video：`app/api/generate/video/route.ts`

常见原因：
- 你传了 `baseUrl` 但没传 `apiKey`，且该分支不再走服务端兜底 key
- `server-providers.yml` / `.env.local` 未配置该能力类型下的 key
- providerId 不匹配配置项（例如配置的是 `openai-tts`，请求传成别的）

处理建议：
- 确认 providerId 和配置分区一致：`tts`/`asr`/`image`/`video`。

### 2.3 `INVALID_URL` / SSRF 相关

第一定位点：`lib/server/ssrf-guard.ts`
第二定位点：各 route 中 `validateUrlForSSRF` 调用

常见原因：
- 生产环境自定义 `baseUrl` 被 SSRF 规则拦截
- baseUrl 不是合法公网地址或命中内网/回环限制

处理建议：
- 先在开发环境验证，再按安全策略调整部署端可访问地址。

### 2.4 `TRANSCRIPTION_FAILED`

第一定位点：`app/api/transcription/route.ts`
第二定位点：`lib/audio/asr-providers.ts`

常见原因：
- providerId 不支持或 modelId 与服务不匹配
- 音频为空/太短/格式不兼容
- 私域 OpenAI 兼容 ASR 接口 path 或协议不一致

处理建议：
- 先用最小样本音频测试（3-10 秒清晰语音）
- 核对服务端是否兼容 OpenAI transcription 请求格式

### 2.5 `GENERATION_FAILED`（TTS）

第一定位点：`app/api/generate/tts/route.ts`
第二定位点：`lib/audio/tts-providers.ts`

常见原因：
- tts provider 选错，voice/model 与该 provider 不兼容
- 私域 baseUrl 正确但接口 path 不兼容（如 `/audio/speech`）
- key 可用但服务返回业务错误（格式/参数超限）

处理建议：
- 用同一 key/baseUrl 在 curl 或 Postman 先验证最小请求可通。

### 2.6 `CONTENT_SENSITIVE`

第一定位点：
- `app/api/generate/image/route.ts`
- `app/api/generate/video/route.ts`

说明：
- 这是上游内容安全策略拦截，不是你平台代码崩溃。

处理建议：
- 调整 prompt（去除敏感词、过激描述），再重试。

### 2.7 `UPSTREAM_ERROR`（verify 接口）

第一定位点：
- `app/api/verify-image-provider/route.ts`
- `app/api/verify-video-provider/route.ts`
- `app/api/verify-model/route.ts`

常见原因：
- key/baseUrl/model 任一不匹配
- 上游服务可达但鉴权失败

处理建议：
- 先通过 verify 接口打通，再接生成接口，别一步到位排查。

### 2.8 `Classroom generation job not found`

第一定位点：`app/api/generate-classroom/[jobId]/route.ts`
第二定位点：`lib/server/classroom-job-store.ts`

常见原因：
- jobId 格式不合法或拼错
- 任务文件被清理或生成失败未写入

处理建议：
- 先确认 `POST /api/generate-classroom` 返回的 jobId 原值是否被前端改写。

### 2.9 任务一直 `running` 不结束

第一定位点：`lib/server/classroom-generation.ts`
第二定位点：`lib/server/classroom-job-store.ts`（stale 逻辑）

常见原因：
- 某一步上游调用阻塞
- 进程重启导致 job runner 丢失
- 媒体生成阶段过慢

处理建议：
- 看进度 `step` 卡在哪个阶段，再去对应模块查。

## 3. 能力维度快速定位

### 3.1 语言模型（LLM）

先看：
- `lib/server/resolve-model.ts`
- `lib/server/provider-config.ts`
- `app/api/generate/scene-outlines-stream/route.ts`

关注点：
- `x-model/x-api-key/x-base-url` 是否生效
- providerId 与 key 映射是否一致

### 3.2 语音合成（TTS）

先看：
- `app/api/generate/tts/route.ts`
- `lib/audio/tts-providers.ts`

关注点：
- `ttsProviderId/ttsModelId/ttsVoice`
- 私域是否兼容对应 provider 的 path/body

### 3.3 语音识别（ASR）

先看：
- `app/api/transcription/route.ts`
- `lib/audio/asr-providers.ts`

关注点：
- `providerId/modelId/language`
- 音频内容与格式

### 3.4 图像/视频

先看：
- `app/api/generate/image/route.ts`
- `app/api/generate/video/route.ts`
- `lib/media/image-providers.ts`
- `lib/media/video-providers.ts`

关注点：
- header 的 `x-image-model/x-video-model` 是否传到位
- provider 适配器是否支持你的私域协议

## 4. 私域 OpenAI 兼容模型排错模板（直接套）

1. 明确你对接的是哪一类能力：LLM / TTS / ASR / Image / Video。
2. 在该能力 route 中确认：
- 用的是 body 参数还是 header 参数
- model 字段名是什么
- baseUrl 分支是否绕过服务端 key 兜底
3. 在 provider 实现中确认：
- 实际请求 path 是什么
- body 字段是否与你私域要求一致
- 返回解析字段是否一致
4. 先打 verify（若有），再打 generate。

## 5. 推荐你常用的 3 个调试动作

1. 先调 verify 接口做连通性验证。
2. 把失败请求保存成最小 JSON/表单样例，固定复现。
3. 每次只改一个变量（providerId 或 baseUrl 或 model），避免混改。
